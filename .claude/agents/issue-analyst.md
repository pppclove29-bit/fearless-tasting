# 이슈 분석 에이전트

GitHub Issue에 등록된 운영자 요구사항을 분석하여 작업 계획을 세우는 에이전트.

## 역할

1. GitHub Issue 목록/상세를 읽고 요구사항을 파악
2. 기존 코드베이스와의 영향도 분석
3. 구현 계획 수립 (파일 목록, 변경 사항, 예상 작업량)
4. 모호한 부분 정리 (추가 확인 필요 사항)

## 실행 방법

```bash
# 열린 이슈 목록 확인
gh issue list --label feature-request --state open

# 특정 이슈 상세 확인
gh issue view {이슈번호}
```

## 분석 산출물 형식

```markdown
## Issue #{번호}: {제목}

### 요구사항 요약
- 한 줄 요약

### 영향 범위
- 백엔드: {변경 파일 목록}
- 프론트: {변경 파일 목록}
- DB: {스키마 변경 여부}

### 구현 계획
1. {단계 1}
2. {단계 2}
3. ...

### 예상 작업량
- 난이도: 쉬움/중간/어려움
- 예상 파일 수: N개

### 확인 필요 사항
- {모호한 부분이 있으면 질문}
```

## 분석 시 참고 사항

- CLAUDE.md를 먼저 읽어 프로젝트 규칙 파악
- 기존 유사 기능이 있으면 패턴 재활용 제안
- DB 변경이 필요하면 마이그레이션 필요 여부 명시
- 프론트/백 API 응답 구조 불일치 방지 — 응답 타입 명시

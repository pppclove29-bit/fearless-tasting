# 무모한 시식가 — 제품 정책 및 기획 문서

## 1. 서비스 개요

**무모한 시식가**는 친구, 연인, 가족끼리 비공개 방을 만들어 맛집 리스트를 함께 기록하고 공유하는 프라이빗 맛집 리뷰 플랫폼입니다.

- **도메인**: https://musikga.kr
- **인증**: 카카오 · 네이버 OAuth (JWT Bearer + localStorage)
- **호스팅**: Cloudflare Pages (프론트) + Render (백엔드) + Cloudflare R2 (이미지)

## 3. 인증 정책

### OAuth 흐름 (카카오 · 네이버)

```
/login 페이지에서 프로바이더 선택
  → /auth/kakao 또는 /auth/naver (백엔드)
  → 인가 코드 → 프로바이더 토큰 → 유저 정보 조회
  → JWT 발급 → /login?access_token=...&refresh_token=... 리다이렉트
  → 프론트에서 localStorage 저장 후 원래 경로(`login_redirect`) 이동
```

- 최초 로그인 시 `Account` 레코드 생성 (`provider: 'kakao' | 'naver'`)
- 기존 유저가 다른 프로바이더로 로그인하면 이메일 기반으로 Account 연결(정책 확인 필요)

### 토큰 정책

| 토큰 | 만료 | 저장 |
|------|------|------|
| Access Token | 15분 | localStorage |
| Refresh Token | 7일 | localStorage + DB (bcrypt 해시) |

### 토큰 갱신

- Access Token 만료 **1분 전**에 프론트에서 선제 갱신
- 401 응답 시 Refresh Token으로 재시도 → 실패 시 로그아웃
- 동시 401 발생 시 refresh 요청 **1회만** 실행 (mutex)

### 로그아웃

1. FCM 토큰 서버에서 삭제
2. localStorage 토큰 삭제
3. 서버 DB에서 refresh token 무효화 (fire-and-forget)

## 5. 이미지 업로드 정책

### 아키텍처 (Presigned URL)

```
프론트: 파일 선택 → 브라우저에서 압축 → 서버에 URL 요청
서버: presigned PUT URL 발급 (5분 유효)
프론트: R2에 직접 업로드 (서버 부하 0) → imageUrl을 API에 전달
```

### 압축 정책 (프론트 — browser-image-compression)

| 항목 | 값 |
|------|---|
| 최대 파일 크기 | 500KB |
| 최대 해상도 | 1200px |
| 포맷 | WebP 변환 |
| 처리 | Web Worker (메인 스레드 안 막음) |

### 폴더 구조 (R2)

```
fearless-tasting-images/
├── profiles/{userId}.webp              # 프로필 (덮어쓰기)
├── reviews/{userId}/{ts}-{hash}.webp   # 리뷰 사진 (최대 3장)
└── restaurants/{restaurantId}.webp     # 식당 대표사진 (덮어쓰기)
```

### 허용 형식

- `image/jpeg`, `image/png`, `image/webp`만 허용
- 그 외 형식은 400 에러

### 정리 정책

| 시점 | 처리 |
|------|------|
| 리뷰 삭제 | 해당 리뷰 이미지 R2에서 삭제 |
| 회원 탈퇴 | 프로필 + 전체 리뷰 이미지 삭제 |
| 프로필 변경 | 같은 키(`profiles/{userId}.webp`)로 덮어쓰기 |

### Rate Limit

- presigned URL 발급: **20회/분**

- **발송**: DB 알림 저장 후 fire-and-forget으로 FCM 발송
- **수신 설정**: 유저별 `pushEnabled` 토글 (프로필에서 on/off)
- **다기기 지원**: 유저당 여러 FCM 토큰 등록 가능
- **만료 토큰**: 발송 실패 시 자동 정리 (UNREGISTERED, INVALID_ARGUMENT)

- 헤더 벨 아이콘 → 드롭다운 (최근 20개)
- 안 읽은 알림 빨간 뱃지
- 포그라운드 수신 시 뱃지 자동 갱신
- 백그라운드 수신 시 네이티브 알림 → 클릭 시 해당 방으로 이동

### 초대 링크 (멤버 초대)

- **복사 시 문구**: `🍽️ "방이름" 맛집 리스트에 초대합니다!\n우리만의 맛집을 함께 기록해요\n\n{URL}`

### 공개 방 (비로그인 열람)

- **방장(owner)만** 공개 전환 가능 (기본값 `isPublic = false`)
- 토글 전 **경고 모달** (공개 시: 프라이버시·SEO 인덱싱 고지 / 비공개 시: 기존 링크 404 danger 경고)
- 비로그인 사용자도 식당/리뷰/세부 평점 열람 가능 (쓰기 불가)
- **멤버 닉네임·프로필 미노출**, 리뷰 작성자 정보 미노출 (개인정보 보호)
- 목록 노출 **품질 필터**: 식당 3+ · 리뷰 3+ · 90일 내 활동
- `SharedRoomDetail.summary` 포함 (식당 수·리뷰 수·평균 평점·대표 카테고리·대표 지역)
- 식당별 `avgRating` 함께 노출 (SEO/신뢰도)
- 경로: `/rooms/public`(목록), `/rooms/public/:id`(상세)
- 공유 UX: 방 설정 공유 팝오버에 "공개 방 링크" 섹션 (URL 복사 + 카카오톡 리치 공유 버튼)

# B2B 피벗 설계서

**상태**: 초안 · 검토 필요
**작성일**: 2026-04-20
**목적**: "친구·연인·가족 맛집 공유" B2C 서비스에 B2B 수익 레이어를 추가해 지속 가능한 수익 구조를 확보

---

## 1. 왜 B2B인가?

### 현재 한계
- 광고(AdSense)만으론 소규모 방 기반 DAU 수익성 한계
- 공개 방은 SEO/유입 훅일 뿐 직접 매출 없음
- 수익 의존도가 한쪽(광고)에만 몰려 있음

### B2B 기회
- **회식 맛집 정리**는 회사·동호회·스타트업 팀의 실질 페인
- 이미 구현된 기능(방, 투표, 통계, 공유)이 그대로 "팀 도구"로 재활용됨
- 추가 개발비 최소 + 고객당 ARPU 훨씬 높음 (월 구독 기반)

## 2. 타겟 세그먼트

| 세그먼트 | 규모 | 니즈 | 예시 고객 |
|----------|------|------|-----------|
| **🏢 기업 회식 관리** | 20~200명 | 회식 장소 정하기, 비용 관리 용이, 임원 접대 맛집 DB | 스타트업 팀, 중소기업 팀 단위 |
| **🎉 동호회 / 커뮤니티** | 30~500명 | 단체 모임 장소, 지역 정보 공유, 평점 기반 추천 | 테니스 동호회, 독서 모임, 교회 소그룹 |
| **🏫 교육 / 기관** | 50~1000명 | 학생 활동, 행사용 식당 조회 | 대학 동아리, 공공기관 복지 담당 |

### 공통점
- 프라이버시(외부 미공개)가 핵심 요구
- 다인 방(현재 20명 제한 → 100+)이 필요
- 관리자 권한 분리 필요

## 3. 가격 모델 (3-tier)

| 플랜 | 가격 (월) | 방 인원 | 방 개수 | 주요 기능 |
|------|-----------|---------|---------|-----------|
| **Free** (현재 B2C) | ₩0 | 20명 | 무제한 | 기본 기능, 광고 노출 |
| **Pro** (개인/커플) | ₩2,900 | 20명 | 무제한 | 광고 제거, 고급 통계, 공유 카드 커스텀 |
| **Team** (소규모 조직) | ₩19,900~ | 100명 | 무제한 | Pro + 멤버 권한 확장, 대시보드, 비용 관리, SSO (미래) |
| **Enterprise** | 별도 협의 | 무제한 | 무제한 | Team + 데이터 export, 감사 로그, 커스텀 도메인 |

### 연간 할인
- Pro 연간: ₩29,000 (약 17% 할인)
- Team 연간: ₩199,000 (약 17% 할인)

### 결제 수단
- **국내**: 토스페이먼츠 (카드 + 계좌이체)
- **해외**: Stripe (향후)

## 5. 단계별 로드맵

### Phase 1 — 결제 인프라 + Pro 출시 (약 4~6주)
- **목표**: 광고 제거 Pro 플랜으로 월 구독 수익 시작
- **작업**:
  - Room 모델: `plan` 필드 (`'free' | 'pro' | 'team'`), `billingOwnerId`, `subscriptionId`
  - 유저별 구독 상태 관리 (`Subscription` 모델 신설)
  - 토스페이먼츠 결제 연동 (등록·해지·갱신 웹훅)
  - Pro 가입자 UI: 광고 숨김, 고급 통계 잠금 해제
  - `/pricing` 페이지 신설

### Phase 2 — Team 플랜 출시 (약 6~8주)
- **목표**: 기업·동호회 대상 다인 방 + 관리 기능
- **작업**:
  - 방 인원 제한 확장 (100명)
  - 조직 대시보드 (`Organization` 모델 도입, 방들을 묶는 상위 개념)
  - 멤버 초대 승인제 (owner/admin이 수락)
  - 비용 관리: RoomVisit에 `costAmount` 필드 + 방별 합계 리포트
  - CSV export API
  - 팀 세일즈 랜딩 페이지 (`/for-teams`) 및 문의 폼

### Phase 3 — Enterprise (고객 확보 후)
- **목표**: 대형 고객 맞춤 지원
- **작업**: SSO, 감사 로그, 커스텀 도메인, SLA
- 실제 계약이 들어올 때 구현 (YAGNI)

## 6. 데이터 모델 변경 (Phase 1)

```prisma
// 구독 (유저·조직 단위)
model Subscription {
  id              String   @id @default(cuid())
  ownerId         String   // User FK (결제 주체)
  owner           User     @relation(fields: [ownerId], references: [id])
  plan            String   @db.VarChar(20)  // 'pro' | 'team'
  status          String   @db.VarChar(20)  // 'active' | 'past_due' | 'canceled'
  provider        String   @db.VarChar(20)  // 'toss'
  providerSubId   String?  @db.VarChar(100) // 토스 빌링키 등
  currentPeriodEnd DateTime?
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  @@index([ownerId, status])
}

// Room에 추가
model Room {
  // ... 기존 필드
  plan            String   @default("free") @db.VarChar(20)
  subscriptionId  String?
  // Team 플랜 멤버 상한은 plan='team' 일 때 100, 그 외 20
}
```

### 기능 분기 로직
- 광고 렌더링: `if (room.plan === 'free') showAd()`
- 멤버 상한: `const max = room.plan === 'team' ? 100 : 20`
- 고급 통계 API: plan 체크 미들웨어로 403 반환

## 8. 리스크 & 가정

### 주요 리스크
1. **결제 도입 기술 부담** — 토스페이먼츠 연동, 환불·해지 처리, 세금계산서 발급
2. **B2B 고객 획득 비용(CAC)** — 기업 영업은 B2C보다 훨씬 길고 비쌈. 초기엔 지인 네트워크 기반 영업 필요
3. **Free 유저 이탈** — 광고 제거 압박 과할 시 브랜드 데미지
4. **프라이버시 우려** — B2B 고객은 data export·감사 등 법적 의무에 민감

### 가정
- 현 DAU가 충분해서 Pro 전환율 1~2%만 되도 의미 있는 MRR 확보 가능
- 한국 시장에서 B2B SaaS 월 ₩20K 수준은 수용 가능한 가격대
- 팀 단위 계약이 하나만 체결돼도 Pro 100명 수준 매출

### 성공 지표 (6개월 목표)
- **Phase 1**: Pro 유료 가입자 100명 이상, MRR ₩290K
- **Phase 2**: Team 플랜 3계약 이상 (약 ₩600K MRR 추가)
